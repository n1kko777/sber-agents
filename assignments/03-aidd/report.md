# Отчёт по проекту «Telegram ESL Tutor Bot»

## Краткое описание
Проект представляет собой Telegram-бота, который с помощью LLM ведёт диалог с пользователем и помогает изучать английский язык. Решение построено по принципам KISS и YAGNI: только базовые сценарии, необходимые для проверки гипотезы.

## Роль ИИ-ассистента
- **Выбранная роль:** Преподаватель английского языка в Telegram.
- **Обоснование:** Пользователю нужен быстрый доступ к подсказкам и объяснениям по английскому без установки дополнительных сервисов. Роль преподавателя позволяет адаптировать ответы под учебный контекст и тон общения.

## Реализованные возможности
- [x] Базовый Telegram-бот на aiogram с polling.
- [x] Интеграция с OpenRouter через openai client и системный промпт преподавателя.
- [x] Память на основе in-memory стора для краткого контекста диалога.
- [x] Обработка стартовых команд, fallback при ошибках, структурированное логирование и проверка конфигурации.

## Технологический стек
- Python 3.11
- uv (управление зависимостями и запуском)
- aiogram 3 (polling)
- openai python client (OpenRouter)
- make

## Инструменты AI-driven разработки
- **AI-coding IDE:** Codex CLI.
- **LLM для генерации документации и кода:** OpenAI Codex (GPT-5) в составе Codex CLI.
- **Модель, используемая приложением:** `openai/gpt-oss-20b:free` (через OpenRouter API).

## Процесс разработки: вызовы и решения
- Сохранение простоты архитектуры — ограничились минимумом модулей (config, llm, memory, main), чтобы ускорить итерации.
- Учёт контекста без БД — использовали `deque` в памяти, что удовлетворяет требованию демо и не требует внешних сервисов.
- Обработка ошибок LLM — добавлен fallback-ответ и логирование через `logger.exception`, чтобы не прерывать сессию пользователя.
- Контроль конфигурации — проверка переменных в `Makefile` и логирование публичной информации конфигурации для диагностики.

## Что нового узнал в процессе AI-driven разработки
- Как организовать пошаговую работу с AI-ассистентом по заранее согласованному таск-листу.
- Практику согласования решений с отрывками кода до каждой итерации — снижает количество переделок.
- Пользу явных правил (conventions + workflow) для передачи контекста код-ассистенту.
- Как быстро разворачивать Python-проект с `uv` и структурировать команды через `make`.
- Поддержание наблюдаемости в прототипе за счёт минимального логирования и проверки конфигурации.
