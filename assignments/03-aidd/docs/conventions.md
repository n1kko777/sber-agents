## Правила для генерации кода

- Держись принципов KISS и YAGNI: только необходимый функционал для текущего сценария.
- Используй инструменты и зависимости, перечисленные в `docs/vision.md`; новые технологии вводим только после согласования.
- Пиши явные, короткие функции и простые модули; избегай лишних абстракций и метапрограммирования.
- Говорящие имена важнее комментариев; если логика нетривиальна, добавь короткое пояснение.
- Конфигурацию читай из переменных окружения и передавай явно в компоненты.
- Логи веди через стандартный `logging`, достаточно уровней INFO и ERROR.
- Структуру файлов держи плоской и прозрачной: модули для бота, LLM и конфигурации отделены, но без глубоких вложенностей.
- Разработку и проверки запускай через цели `make`; никаких кастомных скриптов без необходимости.
