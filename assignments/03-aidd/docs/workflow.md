## Workflow по tasklist

1. Перед стартом итерации прочитай `docs/tasklist.md` и обсуди план с описанием шагов и примером ключевого кода.
2. Дождись подтверждения и только потом реализуй согласованный функционал.
3. По завершении покажи изменения, дождись подтверждения и отметь задачи в тасклисте.
4. Обнови таблицу прогресса в `docs/tasklist.md`, затем зафиксируй результат коммитом.
5. Переходи к следующей итерации только после явного согласия.
