# AI-ассистент клиентского менеджера банка

## Описание проекта
Интеллектуальный ассистент помогает клиентскому менеджеру банка оперативно находить ответы на вопросы, используя RAG-подключение к методическим материалам по рабочим инструментам. Менеджер задаёт вопросы голосом либо текстом, ассистент формирует структурированный ответ, указывает релевантные выдержки из документов и подсвечивает разделы с изображениями, полезными для решения задачи.

## Целевая аудитория
- Клиентские менеджеры и сотрудники фронт-офиса, которые консультируют клиентов и работают с внутренними банковскими системами.
- Новые сотрудники, проходящие адаптацию и обучение инструментам обслуживания.
- Наставники и методисты, которым нужна быстрая проверка актуальности инструкций.

## Основные функции и возможности
- Голосовой ввод запросов с автоматической транскрибацией в текст и контекстной очисткой.
- Ответы на вопросы по базе знаний с цитированием релевантных фрагментов и ссылками на изображения в методических документах.
- Формирование структурированных подсказок (шаги, предупреждения, ссылки) для применения в рабочей системе.
- Режим уточняющих вопросов, если ассистенту не хватает контекста из RAG-хранилища.
- История диалогов для повторного обращения и анализа типовых ситуаций.

## Технологии и подходы из курса
- **Работа с LLM через API:** OpenAI-совместимый API для генерации ответов, классификации намерений и определения шага сценария.
- **Structured Output:** Стандартизованные JSON-ответы с полями «диагноз», «шаги», «ссылки» для интеграции в рабочие чек-листы.
- **Мультимодальность:** Загрузка методических материалов с изображениями в хранилище; генерация текстовых описаний к изображениям и отображение релевантных иллюстраций в ответах.
- **RAG:** Индексация методических материалов, включая изображения (через подписи и описания), и выполнение retrieval на их основе с последующим обогащением запросов.
- **Агентные паттерны и LangGraph:** Построение агентного workflow с ветвлением по типу запроса (инструкции, проблемы, обучение), дообогащение контекста и управлением диалогом.
- **MCP (Model Context Protocol):** Подключение внешних инструментов: преобразование речи в текст, поиск по файловому хранилищу документации, генерация выдержек из изображений.

## План развития проекта по темам курса
- **Т01 — Основы LLM и ИИ-агентов:** Определить задачи ассистента, подобрать модели и сформировать базовую архитектуру.
- **Т02 — Разработка ИИ-ассистента с LangChain:** Реализовать простой чат-ассистент с API вызовами и историей диалога.
- **Т03 — AI-driven разработка с Cursor:** Настроить рабочий процесс разработки, оформить первые сценарии и тестовые запросы.
- **Т04 — Мультимодальные возможности:** Добавить голосовой ввод через ASR и обработку изображений из методических материалов.
- **Т05 — Технология RAG:** Подготовить хранилище документов, выполнить векторизацию текстов и описаний изображений.
- **Т06 — Создание RAG-ассистента с LangChain:** Интегрировать retrieval в диалог, обеспечить цитирование источников и ссылку на изображения.
- **Т07 — Мониторинг и оценка качества:** Настроить метрики ответов, собирать фидбек менеджеров, внедрить автоматические проверки.
- **Т08 — Агентные паттерны:** Спроектировать роли агента (интервьюер, поисковик, наставник), научить задавать уточняющие вопросы.
- **Т09 — LangGraph:** Построить сложный граф состояний: маршрутизация по типу запроса, fallback-логика, повторная генерация.
- **Т10 — Agents + MCP:** Интегрировать внешние инструменты: конвертация речи в текст, преобразование изображений, доступ к файловым системам.
- **Т11 — Безопасность и оценка:** Ввести политику фильтрации конфиденциальных данных, аудит логов и тесты на корректность ответов.
- **Т12 — Разбор проектов:** Подготовить демонстрацию ассистента, собрать отчёт о внедрённых возможностях и результатах обучения.
